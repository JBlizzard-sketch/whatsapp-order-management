WhatsApp order management for Nairobi SMEs


You are building a full production-grade WhatsApp-native order and inventory management system for small Nairobi retail businesses. Read the full brief, expand the vision, present ~20 substantial phases, wait for "next."
The pain:
Thousands of small businesses in Nairobi — pharmacies, butcheries, grocery shops, electronics stalls, hardware stores, clothing boutiques — already run a significant portion of their orders through WhatsApp. Customers message to ask if something is in stock, to place orders, to ask for prices. The shop owner manages this from their personal WhatsApp, in a single undifferentiated inbox, with no record-keeping, no inventory tracking, no payment confirmation, and no way to handle volume. When three customers order the same last item simultaneously, chaos. When a delivery person needs to be told what to pick — chaos. When the owner needs to know what sold today — they can't. The software to fix this exists in the West but none of it is designed for a Nairobi duka owner's reality.
What you are building:
A system that connects to a business's WhatsApp Business account and turns incoming order messages into structured order records — automatically. The shop owner gets a clean web dashboard showing all open orders, inventory levels, and daily revenue. When an order comes in via WhatsApp, the system auto-replies with a confirmation and an Mpesa payment request. When payment is confirmed, the order moves to "paid" and inventory is decremented. The owner can also broadcast stock availability and promotions to their customer list from the dashboard. Setup should take under 20 minutes. The interface must work on a basic Android phone.
Critical nuances:
WhatsApp Business API has real costs — the pricing model must account for this so it doesn't destroy margins. The AI parsing of order messages must handle Swahili, Sheng, and mixed-language messages — "niletee kilo moja nyama na bread mbili" must parse correctly. Inventory management must handle variants (size, colour, weight). The Mpesa integration must handle the STK push flow including timeout and failure states gracefully. The dashboard must be genuinely simple — not a POS system with 40 features. The core loop is: message comes in → order created → payment requested → payment confirmed → inventory updated → owner notified. Everything else is secondary.
Expand this. Think about: a customer loyalty tracker built from WhatsApp order history; automated low-stock alerts; a simple supplier reorder workflow; multi-staff access so two employees can manage orders simultaneously; a "menu" or "catalogue" that customers can request via WhatsApp and browse; end-of-day sales summary sent automatically to the owner's WhatsApp; and a analytics dashboard showing best-selling items, peak order times, and top customers.
Hosting: Backend on Railway (Node.js, webhook handler for WhatsApp, Mpesa callbacks). Frontend dashboard on Vercel. WhatsApp Cloud API (Meta). Database PostgreSQL. Heavy message processing should be Docker-ready for VPS given potential volume.
